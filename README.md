# スイスイドロー工房

手書き風UIのスイスドロー大会マネージャー。ブラウザだけで動作し、サーバーを必要としません。

## 特徴

- **完全クライアント完結**: データはすべてブラウザの `localStorage` に保存。サーバー不要
- **手書き風デザイン**: 方眼紙背景 + Yomogiフォントで紙に手書きした大会運用表の雰囲気
- **スイスドロー方式のペアリング**: MTG公式と同等のアルゴリズム(バックトラックで再戦回避)
- **詳細な大会設定**:
  - タイブレーク方式 4プリセット(標準 / シンプル / 直接対決優先 / Buchholz系)
  - 勝ち点の自由設定(勝/引分/負/Bye)
  - 初回ペアリング(ランダム / 入力順)
  - 再戦ポリシー(厳密禁止 / 不可避時許可 / 自由)
  - Bye 受取上限(1回まで / 無制限)
  - スコア入力方式(勝敗のみ / ゲーム単位)
  - ドロップ時の挙動設定
- **ラウンド運用の柔軟性**: 目標ラウンドの途中変更・延長・早期終了が可能
- **順位表と統計**: OMW% / GW% / OGW% / SoS / Buchholz / 中央Buchholz / Sonneborn-Berger を自動計算
- **色分け**: 勝ち=赤 / 負け=青 / 引分=黄 で一目で識別
- **JSON エクスポート/インポート**: 大会データのバックアップ・端末間移動
- **モバイル対応**: スマートフォン・タブレットでも操作可能

## 技術スタック

- **Vite 5** + **TypeScript 5**(フレームワーク不使用、約 12KB gzip)
- **Google Fonts**: Yomogi(手書き風)
- **GitHub Actions** による自動ビルド&デプロイ

## 開発

### 必要環境
- Node.js 18 以上
- npm 9 以上

### コマンド

```bash
npm install        # 依存インストール
npm run dev        # 開発サーバー起動 (http://localhost:5173)
npm run build      # 本番ビルド (dist/ へ出力)
npm run preview    # ビルド済み成果物のローカル確認
npm run typecheck  # 型チェックのみ
```

### ディレクトリ構成

```
src/
├── main.ts              # エントリポイント
├── types.ts             # 型定義
├── state.ts             # 状態管理・CRUD
├── render.ts            # ルートレンダリング
├── scoring.ts           # 勝ち点・タイブレーク計算
├── pairing.ts           # ペアリングアルゴリズム
├── dom.ts               # DOM ユーティリティ
├── strings.ts           # UI文言の集約
├── diagrams.ts          # 設定画面の図説
├── style.css            # スタイル
└── views/
    ├── list.ts          # 大会一覧 / TOP
    ├── create.ts        # 大会作成
    └── tournament.ts    # 大会進行(順位表・履歴・参加者管理)
```

## ブランチ戦略 & デプロイ

| ブランチ | 役割 |
|---|---|
| `develop` | 開発用 |
| `main` | リリース用(push で自動デプロイ) |
| `gh-pages` | GitHub Pages 公開用(自動更新) |

### デプロイフロー

1. `develop` で作業
2. `main` への Pull Request を作成 → マージ
3. GitHub Actions が自動で:
   - ビルド(`npm run build`)
   - バージョンタグ付け(SemVer)
   - `gh-pages` へデプロイ

### バージョンタグの自動付与

`main` への push ごとに SemVer タグが自動で付きます:

| マージコミットメッセージ | 動作 |
|---|---|
| 通常 | パッチバンプ(例: `v0.1.3` → `v0.1.4`) |
| `[minor]` を含む | マイナーバンプ(例: `v0.1.4` → `v0.2.0`) |
| `[major]` を含む | メジャーバンプ(例: `v0.2.0` → `v1.0.0`) |

## データの永続化

- すべての大会データはブラウザの `localStorage` に保存されます
- ブラウザを閉じても、PCを再起動しても、**同じブラウザで同じURLを開けば復元**されます
- 以下の場合はデータが失われるため注意:
  - シークレット/プライベートウィンドウを閉じたとき
  - ブラウザの「閲覧データを削除」を実行したとき
  - 別のブラウザ・別の端末で開いたとき
- 大事な大会データは **JSON エクスポート**で外部保存を推奨
